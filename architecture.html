Moneytree Architecture

AionUi operator UX → Moneytree MCP server → LangGraph pipeline → Alpaca MCP execution

Agent Pipeline

  • • AionUi → Moneytree MCP (ticker + thesis)
  • • TA Engine → direction + confidence
  • • Strategist → trade proposal (legs, Greeks, BE)
  • • Risk Mgr → approved + sizing (hard gate)
  • • PM → multi-leg submit + log fill
  • • CLI Runner kept as dev/test path only

Persistence & I/O

  • • Chroma: debate vector memory (RW)
  • • SQLite: fill + PnL trade log
  • • Alpaca MCP Paper: chain · IV · exec
  • • Multi-leg ≤4 legs in one API call
  • • NBBO backtest fills-at-mid optimistic

Risk Gate & Known Traps

  • • Greeks RM rejects breaches, not advisory
  • • Look-ahead: cap dates in every tool call
  • • Daily cadence ≠ 0DTE / weeklies
  • • Defined-risk ≠ naked premium sizing
  • • Pre-check margin before broker submit
